# Adapter - v0 (Component-First)

> Renders `templates/designer-brief.md` into one prompt per component, for
> pasting into v0.

## Rendering Rules

1. Carry the Brand And Constraints Header into a single shared "Design System
   Notes" block, included once at the top of the rendered output. Do not
   repeat it per prompt.
2. For each `## Surface` block in the brief, decompose Layout And Hierarchy,
   Interaction Model, State Coverage, and Deliverables into one component per
   primary behavior, state, or transition. Each component gets its own prompt.
3. Each component prompt states, in order:
   - the component's role within its surface;
   - the states it must render, including empty, loading, error, success,
     permission, offline, disabled, and responsive variants when applicable;
   - the voice and copy it carries (from the header and the surface's
     Guardrails);
   - accessibility, locale, token, and component-reuse constraints;
   - explicit do/don't (Guardrails plus the header's "never be" list);
   - an instruction to compose with existing primitives rather than restyle
     them, when Asset reservations restrict styling.
4. Ask the engine to return the component output plus a short coverage note:
   states covered, tokens introduced or reused, deviations, and questions.
5. Preserve the Derivation Trace as a one-line reference above each prompt
   naming the source surface, so rendered output can be checked back against
   the brief.
6. Leave any "Unresolved Brand Or Creative Calls" out of rendered prompts;
   render only after the inventor gate resolves them.

## Output Shape

```
## Design System Notes
<shared header, once>

## Component: <surface> / <component>
Source surface: <brief surface name>
<prompt text>

Expected return: component output, covered-state checklist, token/component
notes, deviations, and open questions.
```
